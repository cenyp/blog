# npm vs yarn vs pnpm

## npm-v2/v3
1. 广泛使用
2. v2 版本有依赖地域问题
```
|————mod-A@1.0
    |————mod-B@1.0
        |————mod-C@1.0
            |————mod-B@2.0
|————mod-C@1.0
    |————mod-B@2.0
```

## 依赖冲突解决
对于版本不兼容的依赖树，npm的处理是先检查是否版本兼容，如果版本兼容就不重复安装，如果和之前的的传递依赖包版本不兼容，那么就将该依赖包安装至当前引用的包的node_modules下
```
|————mod-A@1.0
|————mod-B@1.0
|————mod-C@1.0
    |————mod-B@2.0
```
